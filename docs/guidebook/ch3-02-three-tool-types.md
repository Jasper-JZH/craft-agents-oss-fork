# 3.2 三类工具——自带、随身、外挂

上一节我们知道了，工具就是 Agent 的"手"。没有手，Agent 只能想，不能做。但你有没有想过——同样是手，差别可大了。你的双手天生就会，你每天背的电脑包是自备的，而电焊机那种专业设备，得专门去租。

Agent 的工具也是这样，分三类：自带的、随身的、外挂的。搞清楚这三类，你就知道一个 Agent 出厂时能干什么，上岗后多了哪些本事，以及它还能怎么扩展。

## SDK 内置工具——你天生就有的双手

想象一下，你刚入职第一天，还没领工牌、没配电脑，但你有一双手——能拿东西、能翻页、能写字。Agent 也一样，它从 Claude Agent SDK 里一"出生"，就自带了一整套基础工具，不需要任何配置，开箱即用：

- **Bash**：执行命令行指令。相当于你打开终端敲命令——装依赖、跑脚本、启服务，全靠它。
- **Read**：读取文件内容。想看某个配置文件写了啥？Read 一调，内容到手。
- **Edit**：编辑文件中的特定内容。比 Write 更精细，只改需要改的那几行，不动其他地方。
- **Glob**：按文件名模式搜索文件。比如你想找所有 `.yaml` 配置文件，`**/*.yaml` 一搜就出来。
- **Grep**：按内容搜索文件。想看哪些代码里用到了 `redis`？Grep 一搜，逐行呈现。
- **WebFetch**：获取网页内容。给个 URL，它帮你把页面抓回来，还能提取关键信息。

除了这几个，还有 Write（写文件）、WebSearch（搜索互联网）、Task（启动子任务）等等。它们共同构成了 Agent 的"天生能力"——任何一个用 Claude Agent SDK 创建的 Agent，都自带这些工具，无需额外操心。

## 会话级工具——每天随身背的工具箱

双手虽然万能，但有些事光靠手还是不够方便。所以你每天上班会带一个工具箱——里面有计算器、U 盘、便签纸之类的常用装备。

Craft Agents 在每个会话（Session）启动时，会自动挂载一批自己实现的工具。它们不是 SDK 自带的，而是 Craft Agents 项目专门开发的，每个会话都自动带上，不用你手动配置：

- **SubmitPlan**：提交计划给用户审批。Agent 想干一件大事之前，先用它把计划列出来，等你点了"同意"才动手。相当于工作中写个方案让老板签字。
- **call_llm**：请另一个 AI 帮忙。当当前 Agent 遇到自己不擅长的领域，可以调用另一个大语言模型来出主意——就像遇到难题时请教同事。
- **spawn_session**：开一个新的对话窗口。任务太多忙不过来？spawn 一个子会话，让"分身"去处理一部分工作。

此外还有 browser_tool（操作浏览器）、source_test（测试数据源连接）、config_validate（检查配置文件）等。这些工具是"随身的"——只要你启动了 Craft Agents 的会话，它们就在工具箱里，随时可用。

## MCP Source 工具——按需租的专业外挂

有些活儿，光靠双手和工具箱搞不定。你要焊东西，得租电焊机；你要拍航拍，得租无人机。这些设备不是你天天用，但需要的时候得能接上。

MCP（Model上下文协议，Model Context Protocol）Source 工具就是这样一类"外挂"。它们来自你连接的外部服务，只有在数据源（Source）配置中引入了对应服务，Agent 才能使用这些工具：

- **mcp__linear__createIssue**：在 Linear 里创建一个问题。项目用 Linear 管任务？Agent 可以直接帮你建 Issue。
- **mcp__gmail__sendEmail**：通过 Gmail 发送邮件。需要通知团队成员？Agent 能直接帮你发。
- **mcp__github__createPR**：在 GitHub 上创建 Pull Request。代码改完了，Agent 帮你一键提 PR。

你可能注意到了，这些工具名长得有点特别——前面都有个 `mcp__` 前缀。这不是随便起的，而是 MCP 工具的命名规则：**mcp\_\_{source}\_\_{tool}**。其中 `{source}` 是数据源的名称，`{tool}` 是该数据源提供的具体工具。看到这个前缀，你就知道这是一个外部挂载的 MCP 工具。

外挂的好处是灵活——你连了什么服务，Agent 就多了什么能力。坏处是需要配置，而且不同的项目可能挂了不同的外挂，同一个 Agent 换个项目，能用的工具就不一样了。

## Mini Agent 模式——轻装上阵

还有一点值得提：不是所有 Agent 都需要全副武装。有些简单任务，派一个"轻量级"的 Agent 去就够了——这就是 Mini Agent（迷你智能体）模式。

在 Mini Agent 模式下，工具被限制在最基本的 6 个：Read、Edit、Write、Glob、Grep、Bash。没有审批流程，没有外部服务，甚至没有 WebFetch。就像派一个实习生去改几个错别字——给他一支笔、一个本子就够了，不用把整个工具箱都背过去。

这种模式适合那些目标明确、范围有限的子任务。工具少了，Agent 反而更专注、更快。

## 小结

回顾一下三类工具：

| 类型 | 类比 | 来源 | 举例 |
|------|------|------|------|
| SDK 内置 | 天生的双手 | Claude Agent SDK 自带 | Bash、Read、Grep |
| 会话级 | 随身工具箱 | Craft Agents 项目实现 | SubmitPlan、call_llm、spawn_session |
| MCP Source | 按需租的外挂 | 外部服务配置 | mcp__linear__createIssue、mcp__github__createPR |

从自带到随身再到外挂，工具的能力越来越强，但需要的配置也越来越多。理解这个分类，你就知道一个 Agent 到底"能干多少事"，以及"还能扩展到什么程度"。
