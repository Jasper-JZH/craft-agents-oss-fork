# 5.1 无头服务器——没有界面也能跑 Agent

前四章我们从引擎到工具到子进程一路深入，了解了 Agent 怎么思考、怎么动手、怎么安全地运行。但到目前为止，所有讨论都围绕着一台电脑、一个桌面应用——你坐在电脑前，打开 Craft Agents，跟 Agent 对话。可是，如果你关上电脑、离开工位，Agent 还能继续工作吗？

想象一台自动售货机。它没有店员站在后面递饮料，也没有橱窗装饰吸引路人，但你投币、按键，饮料就会掉出来——交易照样完成。无头服务器（Headless Server）就是这样：没有图形界面、没有按钮和窗口，但它能运行 Agent、接收指令、执行任务，一切照常运转。

## 什么是无头服务器？

"无头"（Headless）这个词听起来有点吓人，但意思很简单——只有"身体"没有"头"。在计算机世界里，"头"指的是图形界面（Graphical User Interface，GUI），也就是你看到的窗口、按钮、菜单这些东西。"身体"指的是后端逻辑——程序的核心功能。

所以，无头服务器就是只运行后端逻辑、不提供图形界面的服务器程序。它像一台 ATM 机——没有银行柜台、没有微笑的柜员，但你能取钱、转账、查余额，所有核心功能都在。你不需要看到 ATM 的"脸"，只需要知道怎么跟它交互。

在 Craft Agents 的语境下，无头服务器意味着：Agent 不一定非要在桌面应用里跑。它可以跑在远程服务器上，而你的桌面应用只是一个"遥控器"，负责显示结果和发送指令。

## 为什么需要无头模式？

你可能会问：桌面应用好好的，干嘛要搞什么无头服务器？原因很实际：

### 1. 远程访问

把 Agent 跑在云服务器——VPS（Virtual Private Server，虚拟专用服务器）上，你就能从任何设备连接它。手机、平板、公司的旧电脑、网吧的机器……只要能装桌面应用或者打开浏览器，就能用你的 Agent。你的 Agent 不再被绑死在某台特定的电脑上。

### 2. 长时间运行

你的笔记本电脑不可能 24 小时开着，但服务器可以。如果你让 Agent 做一个耗时很长的任务——比如监控某个数据源的变化、定时生成报告、持续处理消息——你总不能为了它一直不关机吧？无头服务器跑在云端，7x24 在线，Agent 的会话不会因为你合上笔记本而中断。

### 3. 计算能力

大模型推理是吃资源的大户。你的笔记本可能跑一个 Agent 就已经风扇狂转了，要是同时跑三四个呢？云服务器的 GPU 和 CPU 比笔记本强得多，适合处理重任务。把计算密集的工作丢到服务器上，你的本地机器就可以轻装上阵。

### 4. 团队协作

一个无头服务器可以被多人同时连接。团队成员可以共享同一个 Agent 会话（Session），共享同一套数据源（Source），协作处理任务。这比在每个人自己的电脑上各自跑一份要高效得多——数据不分散、配置不重复。

### 5. CI/CD 集成

CI/CD（Continuous Integration / Continuous Deployment，持续集成/持续部署）流水线是现代开发的标准操作。在自动化流程中，你可能需要 Agent 自动审查代码、生成变更日志、执行测试。这些场景下没有人在电脑前操作界面，Agent 必须以无头模式运行——被脚本调用、被流水线触发，安安静静地把活干完。

## Craft Agents 的无头架构

了解了"为什么"，再来看"怎么做"。Craft Agents 的无头架构可以分成两端：

**服务器端**——代码在 `packages/server/` 里，运行在远程机器上（比如一台 Linux VPS）。它负责所有重活：运行 Agent、执行工具、调用 LLM（Large Language Model，大语言模型）、管理会话状态。服务器没有图形界面，启动后在后台默默工作。

**客户端**——也就是你桌面上的 Craft Agents 应用。但它在这里变成了一个"瘦客户端"（Thin Client）。什么叫瘦客户端？就是它只负责两件事：渲染 UI（把界面画出来给你看）和转发操作（把你点的按钮、打的字发给服务器）。所有真正的逻辑——Agent 怎么思考、工具怎么执行、LLM 怎么调用——全在服务器上完成。

两者之间通过 WebSocket（网页套接字）连接。WebSocket 是一种保持长连接的通信协议，就像一条始终畅通的电话线——服务器随时可以把新消息推过来，客户端随时可以把指令发过去，不需要反复拨号。

用一个比喻：服务器是厨房，客户端是餐桌。厨房里炒菜、切配、调味（运行 Agent、执行工具、调用 LLM），餐桌上只负责摆盘和展示（渲染 UI）。你点一道菜（发送指令），厨房做好了端上来（推送结果），你不需要走进厨房，也不需要知道灶台怎么用。

## 快速启动：一行命令跑起来

理论讲完了，来点实际的。启动一台无头服务器，只需要一行命令：

```bash
CRAFT_SERVER_TOKEN=$(openssl rand -hex 32) bun run packages/server/src/index.ts
```

拆开来看：

- `CRAFT_SERVER_TOKEN=$(openssl rand -hex 32)`——生成一个 64 位随机字符串作为认证令牌（Token）。这个 Token 是连接服务器的"钥匙"，没有它，谁也连不上你的 Agent。
- `bun run packages/server/src/index.ts`——用 Bun 运行服务器的入口文件。

服务器启动后，会打印两行关键信息：

```
CRAFT_SERVER_URL=ws://127.0.0.1:9100
CRAFT_SERVER_TOKEN=a1b2c3d4e5f6...（你生成的随机令牌）
```

拿到这两个值，打开桌面应用，在连接设置里填入 URL 和 Token，点连接——你的桌面应用就变成了那台远程服务器的"遥控器"。从此以后，你在本地输入的每一句话，都会通过 WebSocket 传到服务器上处理；Agent 的每一次回复，都会从服务器推送到你的屏幕上。

如果你想让服务器监听外部地址（比如 VPS 的公网 IP），需要设置 `CRAFT_RPC_HOST` 环境变量；为了安全，Craft Agents 默认拒绝在没有 TLS（Transport Layer Security，传输层安全）加密的情况下绑定公网地址——毕竟 Token 是你的"钥匙"，明文在网上传输可不行。配置 TLS 只需要提供证书和密钥的路径，服务器就会自动切换到加密的 `wss://` 连接。

这就是无头服务器的全貌：没有界面，但能干所有活；跑在云端，但随时随地可以连。不过，客户端和服务器之间到底是怎么"对话"的？这条 WebSocket 上跑的到底是什么格式的消息？下一节我们就来拆解 WebSocket RPC 的通信机制。
