# 3.1 什么是"工具"？Agent 怎么使用工具？

上一章我们了解了 Agent 的引擎设计——ClaudeAgent 和 PiAgent 各有擅长，双引擎互补让 Agent 拥有了思考的能力。但光会思考还不够。想象一个人，大脑飞速运转，什么主意都想得到，可他没有手——没法开门、没法倒水、没法打字。想法再好，也只能烂在脑子里。

Agent 也一样。没有工具的 Agent，就像一个没有手的人。

## 聊天机器人 vs Agent：一个只会"说"，一个会"做"

你大概率用过聊天机器人——在对话框里问它问题，它给你一段文字回答。你可以让它帮你解释概念、写一篇文章、甚至编个故事。但如果你说"帮我把桌上的文件整理一下"，它只能回你一段整理建议，而不会真的动手。

这就是聊天机器人的边界：**它只能生成文字，不能和外部世界交互。**

Agent 不一样。你说"帮我整理文件"，它会真的去读你的目录结构，按规则重命名文件，把散乱的东西归档好。它能"做"事，而不仅仅是"说"事。中间的差距，就来自于"工具"。

## 工具是什么？

工具（Tool），就是 Agent 的"手"。

更准确地说，工具是一段预定义的程序，Agent 可以在需要时调用它来完成特定任务。比如：

- **Bash 工具**：执行终端命令，就像 Agent 在命令行里敲键盘
- **文件读写工具**：读取或修改文件内容，就像 Agent 在编辑器里翻文件
- **搜索工具**：在代码库里查找关键词，就像 Agent 用了全局搜索

每种工具都有明确的功能范围和输入输出规范。Agent 不需要知道工具内部怎么实现——就像你用微波炉热饭，不需要懂磁控管的原理，按下按钮就行。

## 工具调用的完整流程：餐厅点餐

那 Agent 到底是怎么使用工具的？整个流程可以用"餐厅点餐"来类比：

**第一步：你告诉服务员想吃面**

你发了一条消息给 Agent："帮我看一下项目里有多少个 TypeScript 文件。"这就像你走进餐厅，跟服务员说想吃一碗面。

**第二步：服务员决定去厨房下单**

Agent 接到消息后，会思考：要完成这个任务，我需要用到什么？它决定调用 Bash 工具，执行一条 `find` 命令。这就像服务员听了你的需求，决定去厨房下单——他不会自己煮面，但他知道该找谁做。

**第三步：厨师做面**

工具开始执行。Bash 工具在终端里跑那条命令，拿到输出结果。这一步 Agent 不参与，就像厨师做面的时候，服务员在旁边等着，不会抢锅铲。

**第四步：服务员把面端给你**

工具把执行结果返回给 Agent。Agent 拿到了文件列表和数量，这就好比厨师把面端到出餐口，服务员取走。

**第五步：服务员问"还要加蛋吗？"**

Agent 拿到结果后，会根据情况决定下一步：是直接把答案告诉你，还是需要再调用工具获取更多信息。比如它发现文件数量特别多，可能会再用一个搜索工具看看有没有明显重复的。这就像服务员端上面之后问你"要不要加个蛋？"——可能还有后续动作。

整个过程的关键在于：**Agent 自己并不执行任何操作，它只是决定"该用什么工具"和"传什么参数"。真正的活儿是工具干的。**

## 为什么 LLM 自己做不了？

这涉及到一个根本性的限制：LLM（大语言模型，Large Language Model）只会生成文字。

不管你问它什么，它的能力边界就是"根据输入预测下一个 token（词元）"。它可以输出一段看起来像终端命令的文字，但那段文字不会真的被执行。就像一个只会写菜谱的厨师，菜谱写得再好，菜也不会自己做好端上桌。

工具打破了这道墙。通过 Tool Use（工具调用）机制，LLM 不再只是"说出"该做什么，而是真的"触发"一个动作。模型的输出从单纯的文字回复，变成了对工具的调用指令——程序收到指令后去执行，再把结果送回给模型。这样一来，LLM 就和外部世界连上了线。

## Agent 的核心循环

把上面的流程抽象出来，就是 Agent 的核心运转方式——Agent Loop（智能体循环）：

1. **Think（思考）**：分析当前情况，决定下一步做什么
2. **Act（行动）**：调用工具执行操作
3. **Observe（观察）**：接收工具返回的结果
4. **Think again（再思考）**：根据新信息，决定继续行动还是给出最终回答

这个循环会一直转下去，直到 Agent 认为任务已经完成。简单的问题可能一圈就够了——问"今天星期几"，调用一次工具拿到答案就结束；复杂的问题可能要转很多圈——重构一个功能，要读文件、改代码、跑测试、修 Bug，反复循环。

你可以在脑海中想象一个小人在不停转圈：看一眼 → 动一下 → 看结果 → 再想想 → 再动一下……直到活干完。

## 你在 Craft Agents 里看到的

当你使用 Craft Agents 时，界面上会不时冒出一些提示："正在执行 Bash""正在读取文件""正在搜索代码"——这些就是工具在干活。每一条提示背后，都是 Agent Loop 又转了一圈：Agent 思考之后决定调用某个工具，工具执行完毕返回结果，Agent 拿到结果继续思考。

下次看到这些提示，你可以脑补那个小餐厅的画面：服务员在餐桌和厨房之间跑了一趟，厨师做好了一份菜，服务员正端着盘子走回来。
