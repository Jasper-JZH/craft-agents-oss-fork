# 3.5 call_llm——Agent 调用 Agent 的魔法

上一节我们了解了 MCP 协议，知道了 Agent 怎么通过"外挂"获得外部服务的能力。但有一种场景，外挂帮不上忙——Agent 自己在干活时，碰到了需要"再想一遍"的子任务。比如读完十几个文件后需要写一段摘要，拿到一堆数据后需要分个类。这种时候，Agent 需要的不是更多工具，而是一双"额外的眼睛"。

工作中你也遇到过这种情况吧：自己整理了一上午的资料，写到总结的时候卡壳了，于是转头跟隔壁工位的同事说："帮我看一眼，帮我总结一下。"同事看完，三分钟就给你一段精炼的摘要。你不用把整件事交出去，只需要在关键步骤请人搭把手。

**call_llm** 就是 Agent 的那位"同事"。它是一个会话级工具，让 Agent 在执行任务的过程中，可以再做一次 LLM（大语言模型）调用——专门处理那些不需要完整上下文、但需要一次独立思考的子任务。

## call_llm 是什么

call_llm 是 Craft Agents 在每个会话中自动挂载的工具之一。我们在 3.2 节提到过它，现在来仔细看看。

当一个 Agent 正在执行你的指令时，它可以调用 call_llm，发起一次新的 AI 调用。这次调用是独立的——它不会干扰当前对话的上下文，也不会占用主对话的 Token（令牌）额度。你可以把它理解为"在主对话之外开了一个临时小会"。

典型场景有三类：

- **总结大量文本**：Agent 读了很多文件，请"同事"帮忙提炼要点——"帮我总结一下这些内容"。
- **分类数据**：Agent 拿到一批数据，请"同事"帮忙归类——"帮我分成三类"。
- **提取关键信息**：Agent 收到一封长邮件，请"同事"帮忙找出行动项——"帮我提取要做什么"。

这些任务的共同特点是：它们不需要 Agent 的完整对话历史，只需要当前的输入内容就能独立完成。用 call_llm 处理它们，既省主对话的上下文空间，又省钱——你可以让"同事"用更便宜的小模型来干活。

## Output Format（输出格式）——告诉同事"我要什么口味"

请同事帮忙，总得说清楚你要什么样的结果。你说"帮我总结一下"，同事可能给你一页纸，也可能给你三句话。为了避免这种沟通偏差，call_llm 提供了 6 种预定义的 Output Format（输出格式），就像点外卖时的"口味选项"——你选了"微辣"，厨师就知道该怎么炒。

| 格式 | 返回内容 | 适合场景 |
|------|----------|----------|
| `summary` | 摘要 + 要点列表 + 字数 | 总结文件、会议记录 |
| `classification` | 分类 + 置信度 + 理由 | 给内容打标签、判断类型 |
| `extraction` | 提取项列表 + 数量 | 从文本中提取实体、关键信息 |
| `analysis` | 发现 + 问题 + 建议 | 代码审查、风险评估 |
| `comparison` | 相似点 + 不同点 + 结论 | 对比两个方案、两段代码 |
| `validation` | 是否有效 + 错误列表 + 警告 | 检查格式、校验数据 |

使用时只需要指定一个参数：

```
call_llm({
  prompt: "帮我总结这个文件的要点",
  attachments: ["/src/auth/handler.ts"],
  outputFormat: "summary"
})
```

这样"同事"就知道你要的是摘要格式，会返回 `summary`（摘要）、`key_points`（要点列表）和 `word_count`（字数）三个字段。

## Output Schema（输出结构）——给同事一张表格模板

预定义的 6 种格式覆盖了常见场景，但有时你需要更个性化的输出。比如你想让"同事"帮你提取所有 API 端点，并且每个端点要包含方法、路径和处理函数三个字段——6 种预定义格式里没有完全匹配的。

这时候就可以用 Output Schema（输出结构）。它的思路就像给同事一张表格模板："按照这个格式填"。你用 JSON Schema（一种描述 JSON 结构的语言）定义想要的字段和类型，"同事"就会严格按这个结构返回结果：

```
call_llm({
  prompt: "提取这个文件中的所有 API 端点",
  attachments: ["/src/routes.ts"],
  outputSchema: {
    type: "object",
    properties: {
      endpoints: {
        type: "array",
        items: {
          type: "object",
          properties: {
            method: { type: "string" },
            path: { type: "string" },
            handler: { type: "string" }
          }
        }
      }
    },
    required: ["endpoints"]
  }
})
```

注意，`outputFormat` 和 `outputSchema` 不能同时使用——你不能既选了"微辣"又要自定义配方，二选一就好。

## 附件处理——让同事也看看文件

很多时候，你需要"同事"看的不是你在对话里说的内容，而是一个具体的文件。call_llm 支持通过 attachments（附件）参数传递文件路径，工具会自动读取文件内容，发给"同事"看。

基本用法很简单，直接给文件路径：

```
attachments: ["/src/auth/handler.ts"]
```

对于大文件，你可以指定行范围，只让"同事"看关键部分：

```
attachments: [{ path: "/logs/app.log", startLine: 1000, endLine: 1500 }]
```

附件处理有几个边界限制，都是出于实际考虑：

| 限制 | 值 | 为什么 |
|------|-----|--------|
| 单个文件大小 | 500KB | 太大了"同事"一次看不完 |
| 附件总大小 | 2MB | 一次请人帮忙不能堆太多材料 |
| 单文件最大行数 | 2000 行 | 同上，超过需要指定行范围 |
| 最多附件数 | 20 个 | 一个人一次能消化的文件有限 |

还有一个贴心的细节：工具会自动检测二进制文件（比如编译后的 `.exe`、`.bin`），发现是二进制就跳过并提示，不会把乱码发给"同事"。

## 模型选择——实习生还是资深员工

请人帮忙，事情简单就找实习生，事情复杂就找资深员工。call_llm 也一样，你可以通过 `model` 参数指定用哪个 AI 模型。

工具支持用短名来选择模型，比如：

- **haiku**：快但简单，适合总结、分类这类轻量任务。就像实习生——手脚快，成本低，日常事务足够。
- **sonnet**：平衡型，能力强但也不算贵。就像有几年经验的同事——大多数问题都能处理。
- 更高级的模型（如 opus）则适合深度分析等复杂任务。

如果不指定模型，默认使用 haiku——毕竟请"同事"帮忙的通常是简单子任务，能省钱就省钱。

## Timeout（超时）——同事不能想太久

请人帮忙，总不能等一下午。call_llm 有一个默认的 Timeout（超时）时间：**2 分钟**。如果"同事"在这个时间内没有给出回复，调用就会超时中断，返回错误信息。

这个时间对绝大多数子任务来说绰绰有余，也防止了因模型卡住而让整个 Agent 流程停摆的情况。

## 后端契约——每个后端都必须实现 queryLlm()

call_llm 看起来是"调一次 AI"这么简单的操作，但背后涉及认证、会话管理、模型路由等细节。不同后端（ClaudeAgent、CodexAgent、CopilotAgent）的实现方式各不相同。不过，它们都必须满足同一个契约：**实现 `queryLlm()` 方法**。

这是 AgentBackend 接口的强制要求。`queryLlm()` 接收一个 `LLMQueryRequest`（包含 prompt、model、systemPrompt 等参数），返回一个 `LLMQueryResult`（包含 text、model、Token 用量等结果）。每个后端用自己擅长的方式来实现这个方法——ClaudeAgent 用 OAuth 认证调 SDK，CodexAgent 用 OpenAI 的线程，CopilotAgent 用临时会话——但对外行为一致。

这种"统一接口、各自实现"的设计，让 call_llm 在任何后端上都能工作，而你不需要关心底层的差异。

## 小结

| 特性 | 类比 | 说明 |
|------|------|------|
| call_llm 本身 | 请同事帮忙 | 在执行中发起一次独立 AI 调用 |
| Output Format | 点外卖选口味 | 6 种预定义格式，一句话搞定 |
| Output Schema | 给同事表格模板 | 自定义 JSON 结构，精准控制输出 |
| 附件 | 让同事看文件 | 支持路径和行范围，有大小限制 |
| 模型选择 | 选实习生还是资深员工 | haiku 快且便宜，sonnet 平衡，opus 强力 |
| Timeout | 不能想太久 | 默认 2 分钟超时 |
| queryLlm() | 后端契约 | 每个后端必须实现的接口方法 |

call_llm 让 Agent 有了"自我分工"的能力——大事自己扛，小事交给"同事"。但工具虽好，谁来保证安全？一个能执行命令、读写文件、调用 AI 的 Agent，如果没有权限约束，就像一个什么都能做的员工没有审批流程——迟早出事。下一节，我们就来认识 Craft Agents 的权限系统。
