# 1.3 技术栈一览——这些名词都是干什么的？

刚才我们看了项目的目录结构，一整棵树摆在那里，各种文件夹和文件名扑面而来。你可能已经注意到了一堆陌生的英文名词——Bun、Electron、shadcn/ui、MCP……它们到底是干什么的？别急，咱们用一个比喻把它们串起来。

想象一下，Craft Agents 是一家**厨房**。一家能运转的厨房，需要灶台、菜谱规范、餐厅门面、摆盘工具、装修方案、厨房电器，还需要一个保险箱来锁好贵重食材。我们的技术栈，恰好一一对应。

---

**Bun（运行时）= 灶台**

灶台是厨房的基础，没有它什么菜都做不了。Bun 就是一块灶台——它是一个 JavaScript/TypeScript 运行时，负责把代码真正"跑起来"。你也许听过 Node.js，而 Bun 是后起之秀，启动更快、安装依赖更快、运行也更快。选 Bun 而不是 Node.js，就像厨师选了猛火灶而不是老式煤炉——同样的菜，出活更利索。

**TypeScript（编程语言）= 菜谱的书写规范**

光有灶台不够，你还得写菜谱。TypeScript 就是菜谱的书写规范。它在 JavaScript 基础上加了"类型检查"——要求你写菜谱时标清楚"盐 5 克"而不是含糊地写"加点盐"。这样，在你动手做菜之前（代码运行之前），就有人帮你检查菜谱有没有写错。虽然多写一点标注，但换来更少的翻车。

**Electron（桌面框架）= 餐厅**

菜做好了，得有一个地方让客人坐下来吃。Electron 就是那间餐厅——它把网页变成桌面应用。VS Code、Discord、Slack 桌面版都是它做的。原理很直白：把浏览器内核塞进桌面窗口，让网页应用像原生软件一样运行。Craft Agents 是桌面 App，这间"餐厅"少不了。

**React（UI 框架）= 摆盘工具**

菜端上桌，摆盘好不好看直接影响食欲。React 是一个 UI 框架，专门负责把界面渲染得漂亮、响应流畅。它用"组件"的方式组织界面——按钮、对话框都是组件，像摆盘工具一样一道道摆好。Electron 提供餐厅的壳子，React 负责每一张桌布、每一个菜单的呈现。

**shadcn/ui + Tailwind CSS（组件库 + 样式方案）= 装修方案**

摆盘工具有了，但每次从零雕萝卜花太累。shadcn/ui 是一套现成的高质量 UI 组件库，按钮、弹窗、输入框拿来就用；Tailwind CSS 则是一套样式工具，用简短的 class 名就能写出好看的排版。两者配合就像请了装修团队——墙纸灯饰都配好了，你只管选风格。

**Vite + esbuild（构建工具）= 厨房电器**

厨房里总得有切菜机、烤箱来提高效率。Vite 和 esbuild 就是这样的厨房电器——负责把源代码打包成浏览器能运行的产物。esbuild 用 Go 写的，速度极快，负责主进程打包；Vite 负责渲染进程（React 那一层），开发时还支持热更新——改一行代码界面立刻刷新。

**MCP（Model Context Protocol，模型上下文协议）= 万能插头标准**

现在厨房运作起来了，但你还想接外面的食材供应商——有的送海鲜，有的送蔬菜，有的送调料。问题是每家的接口不一样，怎么接？MCP 就是一套"万能插头标准"：只要供应商按这个标准做插头（MCP Server），你的厨房（Agent）就能直接插上用。Craft Agents 通过 MCP 连接 GitHub、Notion、Slack 等各种外部服务，就像插上一个新电器一样简单。

**Claude Agent SDK = 和 Claude AI 交流的工具包**

厨房需要一位主厨来决策，Claude 就是那位主厨。Claude Agent SDK 是 Anthropic 官方的开发工具包，让程序能方便地调用 Claude 的能力——发送消息、接收流式回复、使用工具、管理多轮对话。Craft Agents 用它驱动 Anthropic 直连和所有兼容 Claude API 的第三方服务。

**Pi SDK = 和多种 AI 交流的工具包**

但客人可能想请不同风格的主厨——有的偏好 Gemini，有的习惯 ChatGPT，还有的用 GitHub Copilot。Pi SDK 就是另一个工具包，专门负责连接这些非 Anthropic 的 AI 提供方。两个 SDK 并肩工作，让 Craft Agents 能同时支持多种 AI 引擎。

**AES-256-GCM = 加密保险箱**

厨房里有些东西必须锁好——供应商的密钥、API Key、OAuth 令牌，绝对不能随便丢在台面上。AES-256-GCM 是一种工业级加密算法（银行也用它），Craft Agents 用它来加密存储你的所有凭证。加密后的文件存在 `credentials.enc` 里，没有主密码谁也打不开——就像厨房角落里那个只有厨师长有钥匙的保险箱。

---

把这些拼在一起，整个"厨房"就清晰了：Bun 是灶台，TypeScript 规范菜谱，Electron 搭起餐厅，React 摆好桌面，shadcn/ui 和 Tailwind 负责装修，Vite 和 esbuild 加速出餐，MCP 接通外部供应商，Claude Agent SDK 和 Pi SDK 分别请来不同的主厨，AES-256-GCM 锁好保险箱。每一层各司其职，整间厨房高效运转。

其中最核心的就是那两位"主厨"——Claude Agent SDK 和 Pi SDK，它们是整个应用的大脑。下一节我们重点聊聊。
