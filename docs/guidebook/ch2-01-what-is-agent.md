# 2.1 什么是"Agent"？为什么需要"后端"？

上一章我们认识了两种 AI 引擎——Claude 和 Pi，知道它们一个专精、一个通用，各有各的本事。但在打开引擎盖之前，我们先回答两个更根本的问题：到底是什么东西在用这些引擎？为什么不能让 AI 模型直接干活，中间非要隔一层"后端"？

## 从聊天机器人到 Agent——能说能做的 AI

你打客服电话的时候，电话那头的人只能帮你查信息、解释规则，但没办法替你操作任何系统。你问他"帮我改一下收货地址"，他说"好的，请您自己登录 App 修改"。这种只能"说"不能"做"的角色，就是典型的聊天机器人（Chatbot）。

但如果一个维修工上门，他不仅会告诉你水管哪里漏了，还会掏出扳手直接修好。能感知问题、选择工具、动手操作——这就是 Agent（智能体）和聊天机器人的本质区别。

在 Craft Agents 的世界里，Agent 不只是陪你聊天的对话伙伴，而是一个能**自主完成任务的行动者**。它的工作方式可以用一个循环来概括：

1. **理解需求**：你发了一条消息，Agent 分析你想要什么。
2. **选择工具**：根据需求，决定用哪个工具——读文件、跑命令、搜索代码，还是调用某个 API。
3. **执行操作**：调用工具，得到结果。
4. **观察结果**：看看工具返回了什么，判断任务是否完成。
5. **继续或完成**：如果还没做完，回到第 2 步；如果做完了，把结论告诉你。

这个循环会一直转下去，直到任务完成或者被你叫停。正是这种"感知—决策—行动—观察"的闭环，让 Agent 从一个只会说话的助手变成了一个能解决问题的搭档。

## 后端——Agent 的引擎

理解了 Agent 是什么，接下来的问题是：同一个 Agent，为什么需要不同的"后端"（Backend）？

想象一个老司机。他开汽油车能上路，开电动车也能上路，开混动车照样能上路。驾驶技术是他的，但每次出行的动力来源取决于车底下的引擎。汽油引擎烧油，电动引擎充电，混动引擎两者兼顾——引擎不同，开车的方式就不同，但司机感受到的是同一套方向盘和踏板。

Agent 就是这个司机，后端就是引擎。后端决定了：

- **用哪个 AI 模型**：是 Claude、Gemini 还是 GPT？不同的模型有不同的能力和风格。
- **怎么连接**：用 API Key 直连，还是走 OAuth 登录？在主进程内直接调用，还是启动一个子进程通信？
- **怎么执行工具**：工具调用的协议是什么？权限检查怎么做？流式响应怎么传回来？

这些细节全部封装在后端里。Agent 本身只关心"我要完成什么任务"，不关心底层用的是哪家 AI、走的什么协议。

## Craft Agents 的两个后端

Craft Agents 目前有两个后端实现，对应上一章介绍的两台引擎：

- **Claude 后端**（`ClaudeAgent`）：专用引擎，只为 Anthropic Claude 服务。它在主进程内直接调用 Claude Agent SDK，认证方式支持 API Key 和 OAuth，响应速度最快。你可以把它想象成特斯拉的专属充电桩——只充一种车，但充得又快又稳。

- **Pi 后端**（`PiAgent`）：通用引擎，支持 Google、OpenAI、GitHub Copilot 等多种 AI Provider。它通过启动一个子进程来运行 Pi SDK，主进程和子进程之间用 JSONL 协议通信。万能充电器——什么车都能充，虽然充法不同，但结果一样。

在源码中，这两个后端分别实现在 `packages/shared/src/agent/claude-agent.ts` 和 `packages/shared/src/agent/pi-agent.ts` 中。它们各自处理认证、工具注册、事件转换等细节，但对外呈现完全一致的行为。

## 工厂模式——你说要什么，就给你什么

那么，系统怎么知道该创建哪个后端？答案是一个叫 `createBackend()` 的工厂函数（Factory Pattern）。

工厂模式就像一家租车公司。你走进去说"我要一辆电动车"，前台就给你一把电动车的钥匙；你说"我要一辆汽油车"，就给你汽油车的钥匙。你不需要知道车停在哪个车位、钥匙怎么配的——你只需要告诉前台你的需求。

在 `packages/shared/src/agent/backend/factory.ts` 中，核心逻辑简洁明了：

```typescript
export function createBackend(config: BackendConfig): AgentBackend {
  switch (config.provider) {
    case 'anthropic':
      return new ClaudeAgent(config);
    case 'pi':
      return new PiAgent(config);
    default:
      throw new Error(`Unknown provider: ${config.provider}`);
  }
}
```

配置里的 `provider` 字段就是你对租车公司说的那句话。填 `'anthropic'`，拿到 ClaudeAgent；填 `'pi'`，拿到 PiAgent。不管拿到的是哪个，它们都是同一种类型——`AgentBackend`，就像不管租到什么车，方向盘和踏板的位置都一样。

这就是后端抽象的核心价值：**上层代码只需要和 AgentBackend 接口打交道，不需要知道底下跑的是谁。** 换引擎？改一个配置就行，其余代码一行不动。
